## 1. MCP (Model Context Protocol) Best Practices

### 현황

MCP는 Anthropic이 2024년 11월 공개한 오픈 표준으로, AI 모델과 외부 도구/데이터소스 연결을 표준화. 2025-2026년 급속도로 채택 확산.

### 핵심 Best Practices

| 원칙 | 상세 |
|------|------|
| **최소 권한 원칙** | MCP 서버는 필요한 최소 권한만 요청. 파일 읽기 서버가 쓰기 권한 불필요 |
| **Tool 명세 명확화** | description 필드에 도구 목적·입력·출력·사이드이펙트 명시 |
| **에러 처리 표준화** | MCP 에러 코드 체계 준수. 상세 에러 메시지로 디버깅 가용성 확보 |
| **상태 비저장 설계** | 서버는 가능한 stateless 유지. 세션 상태는 클라이언트 책임 |
| **인증 분리** | OAuth 2.0 / API 키를 MCP 레이어와 분리. 런타임 주입 |
| **버전 관리** | Tool 스펙 버전 명시. Breaking change 시 새 tool 이름 사용 |

### 현재 프로젝트 적용 상태

```
✅ GitHub MCP: 표준 OAuth 인증 사용
✅ figma MCP: SKILL.md에 auth 분리 문서화
✅ tool description: agents/*.md에 상세 명세
⚠️ 개선 여지: MCP 서버 에러 처리 표준화 추가 검토
```

## 2. Agent Orchestration Patterns

### 2026년 주요 트렌드

```
트렌드 1: Hierarchical Multi-Agent (계층형)
  Orchestrator → Specialized Agents → Tools
  현재 프로젝트: /auto Lead → executor/planner/architect
  상태: ✅ 구현됨

트렌드 2: Parallel Wave Execution (병렬 웨이브)
  독립 태스크를 Wave로 분할, 동시 실행
  현재 프로젝트: HEAVY 모드 병렬 executor
  상태: ✅ phase-2-build.md에 정의됨

트렌드 3: Feedback Loop with Critic (비평 루프)
  Builder-Critic 쌍 패턴으로 품질 자가 검증
  현재 프로젝트: Planner-Critic Loop (HEAVY)
  상태: ✅ phase-1-plan.md에 정의됨

트렌드 4: Contract-Based Phase Interface
  Phase 간 데이터를 명세된 Contract로 전달
  현재 프로젝트: InitContract/PlanContract/BuildContract/VerifyContract
  상태: ✅ SKILL.md에 정의됨

트렌드 5: Agent Zombie Detection (좀비 탐지)
  비정상 종료 에이전트 자동 감지 및 정리
  현재 프로젝트: subagent_zombie_detector.py
  상태: ✅ hooks에 구현됨
```

### 업계 동향 vs 현재 프로젝트

| 업계 트렌드 | 프로젝트 상태 | 격차 |
|------------|:------------:|------|
| Tool-use 에이전트 표준화 | ✅ agents/*.md 명세 | 없음 |
| Multi-modal 에이전트 | ✅ vision.md | 없음 |
| Worktree 격리 실행 | ✅ --worktree 옵션 | 없음 |
| Adaptive Model Routing | ✅ Trivial/Standard/Complex/Critical | 없음 |
| Cost-aware Routing (eco) | ✅ --eco/-2/-3 | 없음 |

**평가**: 현재 프로젝트는 2026년 오케스트레이션 트렌드와 높은 정렬도를 보임.

## 3. Hooks Automation Patterns

### Claude Code Hooks 생태계 현황

Claude Code hooks는 도구 실행 전후에 사용자 정의 로직을 삽입하는 메커니즘. 2025-2026년 고급 활용 패턴 확산.

### 주요 패턴

| 패턴 | 설명 | 현재 구현 |
|------|------|----------|
| **Circuit Breaker** | 반복 실패 감지 → 자동 차단 | circuit_breaker.py ✅ |
| **Branch Guard** | 보호 브랜치 직접 수정 방지 | branch_guard.py ✅ |
| **Zombie Detection** | 비정상 에이전트 자동 정리 | subagent_zombie_detector.py ✅ |
| **Context Recovery** | 컨텍스트 한계 도달 시 복구 | context_limit_recovery.py ✅ |
| **Auto Deploy** | 세션 종료 시 자동 배포 트리거 | auto_deploy.py ✅ |
| **Slack Reporter** | 편집 활동 실시간 Slack 보고 | edit_slack_reporter.py ✅ |
| **Sample Sender** | 일일 샘플 수집 | daily_sample_sender.py ✅ |

### 발견된 이슈: 경로 환경 불일치

```
문제: hooks 명령이 Windows 경로 C:/claude/... 사용
환경: Linux (/home/user/claude)
영향: PreToolUse hook exit code 2 → Bash/Write/Edit/Agent 전체 차단
수정: GitHub commit b51c11e1 (git pull 필요)
```

### Best Practice 권장사항

```
1. 환경 독립적 경로 사용
   현재: 하드코딩된 절대 경로
   권장: 환경 변수 $CLAUDE_ROOT 도입
   예: command: "python3 $CLAUDE_ROOT/.claude/hooks/xxx.py"

2. Hook 실패 격리
   PostToolUse 오류는 경고만 (현재 동작 올바름)
   PreToolUse exit 2만 차단 (현재 동작 올바름)

3. async: true 활용
   긴 작업 (Slack 보고, 샘플 수집)은 async: true로 비차단
   현재: daily_sample_sender.py, edit_slack_reporter.py ✅
```
