## 3. Hooks 최적화 패턴 (2026 최신)

### 웹 검색 결과 요약

| Best Practice | 설명 | 현재 상태 |
|---------------|------|----------|
| PreToolUse < 500ms | 모든 매칭 도구 호출 차단 — 느리면 세션 슬러그 | 구현됨. 성능 측정 미확인 |
| PostToolUse formatter | 파일 편집 후 자동 포맷 — 멱등성 보장 | post_edit_check.js 구현됨 |
| PermissionDenied hook (신규) | v2.1.89+ — 권한 거부 이벤트 처리 가능 | 미구현. 개선 기회 |
| 21개 lifecycle 이벤트 (2026) | SessionStart, Stop, SubagentStop 등 풀 커버리지 | 주요 이벤트 모두 구현됨 |
| async 비동기 실행 | 독립 훅은 async:true 설정으로 병렬 실행 | checklist_updater.py 등 async:true 적용됨 |

### 갭 분석

- PermissionDenied hook 미구현 — 보안 감사 이벤트 추적 가능
- PreToolUse 성능 측정 없음 — 500ms 초과 여부 미확인
