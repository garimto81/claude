# 언어 및 응답 규칙

## 기본 원칙

| 항목 | 규칙 |
|------|------|
| **출력 언어** | 한글 |
| **기술 용어** | 영어 유지 (code, GitHub, API 등) |
| **충돌 시** | 사용자에게 질문 (임의 판단 금지) |

## 잘못된 예

```
❌ "코드를 깃허브에 푸시했습니다"
❌ "에이피아이 호출을 수정하겠습니다"
```

## 올바른 예

```
✅ "코드를 GitHub에 push했습니다"
✅ "API 호출을 수정하겠습니다"
```

## AI 티 제거 규칙 (CRITICAL)

**'한글(영문)' 형식 사용 금지** - AI가 작성한 문서처럼 보이는 패턴을 피합니다.

### 금지 패턴

```
❌ "사용자(User)가 로그인합니다"
❌ "기능(Feature)을 구현했습니다"
❌ "인증(Authentication) 모듈"
❌ "데이터베이스(Database) 연결"
❌ "프로젝트(Project) 구조"
```

### 올바른 표현

```
✅ "사용자가 로그인합니다"           # 한글만 사용
✅ "User가 로그인합니다"             # 영어만 사용 (기술 용어 시)
✅ "기능을 구현했습니다"
✅ "인증 모듈" 또는 "Authentication 모듈"
✅ "DB 연결" 또는 "데이터베이스 연결"
```

### 적용 원칙

| 상황 | 권장 표현 | 예시 |
|------|----------|------|
| **일반 명사** | 한글만 | "사용자", "기능", "화면" |
| **기술 용어** | 영어만 | "API", "JWT", "OAuth" |
| **고유명사** | 영어 원문 | "GitHub", "React", "Supabase" |
| **혼용 필요 시** | 첫 등장만 병기 후 한글로 통일 | "JWT(JSON Web Token)는... 이후 JWT는..." |

### 문서 작성 시 체크리스트

- [ ] `한글(영문)` 패턴 없음
- [ ] 불필요한 영어 병기 없음
- [ ] 기술 용어는 영어 그대로 사용
- [ ] 일관된 용어 사용 (섞어 쓰지 않음)
