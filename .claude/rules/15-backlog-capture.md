# 백로그 캡처 규칙

**트리거**: 사용자가 기능 추가, 수정, 버그 수정을 요청하는 모든 상황

## 핵심 원칙

사용자 요구사항은 수신 즉시 `docs/backlog.md`에 기록한다. 구현 후가 아님.

## 필수 행동

### Step 1: 요구사항 수신 시
1. `docs/backlog.md`의 `## PENDING` 섹션에 항목 추가
2. 항목 형식:

```markdown
### [B-NNN] 제목
- **날짜**: YYYY-MM-DD
- **설명**: 구체적 요구사항
- **수락 기준**: 완료 판단 기준
- **관련 PRD**: docs/00-prd/xxx.prd.md (있으면)
```

### Step 2: 구현 시작 시
1. 해당 항목을 `## PENDING` → `## IN_PROGRESS`로 이동

### Step 3: 구현 완료 시
1. 해당 항목을 `## IN_PROGRESS` → `## DONE` 테이블로 이동
2. 완료일, 관련 커밋/PRD 기록

## 예외
- `!quick`, `!hotfix`: 백로그 기록 스킵 허용 (1줄 수정 등)
- 질문/확인 요청: 백로그 대상 아님
- 인프라/설정 변경 (CLAUDE.md, rules 등): 백로그 대상 아님
