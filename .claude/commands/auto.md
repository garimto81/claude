# /auto - 자동 완성 반복 루프

사용자 개입 없이 작업을 **연속 자동 실행**합니다. 중단 명령이 있을 때까지 계속 진행합니다.

## 개념

```
┌─────────────────────────────────────────────────────────────────┐
│                    자동 완성 루프 (Auto-Completion Loop)         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   요청 입력 ──▶ 자동 실행 ──▶ 완료 ──▶ 다음 작업 탐색 ──┐       │
│        ▲                                              │       │
│        └──────────────── 반복 ◀───────────────────────┘       │
│                                                                 │
│   중단 조건: 사용자 입력 / 모든 작업 완료 / 심각한 에러         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 사용법

```bash
# 기본 실행 (작업 목록 자동 탐색)
/auto

# 특정 작업으로 시작
/auto "로그인 기능 추가"

# 이슈 기반 실행
/auto #123 #124 #125

# 작업 목록 파일 기반
/auto --from tasks.md
```

## 실행 흐름

```
/auto 실행
    │
    ├─ Step 1: 작업 탐색
    │      │
    │      ├─ 대기 중인 GitHub 이슈 확인
    │      ├─ TodoWrite 미완료 항목 확인
    │      ├─ PRD에서 미구현 항목 확인
    │      └─ 작업 우선순위 결정
    │
    ├─ Step 2: 작업 실행 (/work --auto 호출)
    │      │
    │      ├─ 분석 (자동)
    │      ├─ 구현 (자동)
    │      ├─ 테스트 (자동)
    │      └─ 커밋 (자동)
    │
    ├─ Step 3: 완료 확인
    │      │
    │      ├─ 성공 → 완료 리포트 출력
    │      └─ 실패 → 에러 로그 + 다음 작업으로 이동
    │
    └─ Step 4: 반복 판단
           │
           ├─ 다음 작업 있음 → Step 1로 복귀
           ├─ 모든 작업 완료 → 최종 리포트 출력 + 종료
           └─ 중단 신호 → 현재 상태 저장 + 종료
```

## 작업 탐색 우선순위

| 순서 | 소스 | 설명 |
|------|------|------|
| 1 | 명시된 작업 | `/auto "작업"` 으로 직접 지정 |
| 2 | GitHub 이슈 | `help wanted`, `good first issue` 라벨 우선 |
| 3 | TodoWrite | 상태가 `pending`인 항목 |
| 4 | PRD 미구현 | `[ ]` 체크박스 항목 |
| 5 | 코드 TODO | `# TODO:` 주석 |

## 자동 실행 규칙

### 작업별 자동 처리

| 작업 유형 | 자동 실행 내용 |
|----------|---------------|
| 기능 추가 | 분석 → 브랜치 → TDD → 커밋 → PR |
| 버그 수정 | 이슈 분석 → 수정 → 테스트 → 커밋 |
| 리팩토링 | 분석 → 수정 → 테스트 → 커밋 |
| 문서 작성 | 분석 → 작성 → 커밋 |
| 테스트 추가 | 분석 → 테스트 작성 → 실행 → 커밋 |

### 에러 처리

| 에러 유형 | 자동 대응 | 최대 시도 |
|----------|----------|----------|
| 테스트 실패 | 자동 수정 시도 | 3회 |
| 린트 에러 | `--fix` 적용 | 1회 |
| 빌드 실패 | 의존성 재설치 | 2회 |
| 충돌 | 리베이스 시도 | 1회 |
| 심각한 에러 | 로그 저장 + 스킵 | - |

## 중단 방법

| 방법 | 설명 |
|------|------|
| 메시지 입력 | 아무 메시지나 입력하면 현재 작업 완료 후 중단 |
| `중단` | 즉시 중단 (현재 작업 롤백) |
| `일시정지` | 현재 상태 저장 후 중단 (`/auto --resume`으로 재개) |
| Ctrl+C | 강제 종료 |

## 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--max N` | 최대 N개 작업 후 중단 | `/auto --max 3` |
| `--type TYPE` | 특정 유형만 실행 | `/auto --type bug` |
| `--resume` | 이전 세션 재개 | `/auto --resume` |
| `--dry-run` | 실행 없이 계획만 출력 | `/auto --dry-run` |
| `--no-pr` | PR 생성 안함 | `/auto --no-pr` |
| `--no-commit` | 커밋 안함 (변경만) | `/auto --no-commit` |

## 연동 커맨드

| 커맨드 | 연동 |
|--------|------|
| `/work --auto` | 각 작업 실행에 사용 |
| `/commit` | 자동 커밋 |
| `/issue` | 이슈 탐색 및 상태 업데이트 |
| `/debug` | 에러 발생 시 자동 디버깅 |
