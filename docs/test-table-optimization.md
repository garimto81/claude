# 테이블 컬럼 너비 최적화 테스트

이 문서는 Single-Line First 테이블 최적화 기능을 테스트합니다.

---

## 1. 짧은 컬럼 + 긴 컬럼 혼합

| ID | 상태 | 설명 |
|:--:|:----:|------|
| 1 | ✅ | 이것은 매우 긴 설명 텍스트입니다. 테이블 최적화 기능이 적용되면 ID와 상태 컬럼은 좁게, 설명 컬럼은 넓게 표시됩니다. |
| 2 | ⏳ | 또 다른 긴 설명입니다. Single-Line First 전략은 짧은 텍스트에 정확한 너비를 할당합니다. |
| 3 | ❌ | 마지막 행의 설명입니다. |

---

## 2. 모든 컬럼이 짧은 경우

| 번호 | 코드 | 값 | 단위 |
|:----:|:----:|:--:|:----:|
| 1 | A01 | 100 | kg |
| 2 | B02 | 200 | m |
| 3 | C03 | 300 | L |

## 5. 2열 테이블 (Key-Value)

| 항목 | 값 |
|------|-----|
| 프로젝트명 | Claude Code Table Optimizer |
| 버전 | 2.4.0 |
| 최적화 전략 | Single-Line First |
| 테이블 너비 | 510pt (18cm) |
| 최소 컬럼 너비 | 50pt |
| 최대 컬럼 너비 | 340pt |

---

## 결론

위 테이블들은 각각 다른 특성을 가지고 있으며, Single-Line First 최적화가 적용되어:

1. **짧은 텍스트 컬럼**은 한 줄에 표시될 수 있도록 정확한 너비 할당
2. **긴 텍스트 컬럼**은 남은 공간을 유연하게 사용
3. **한글 문자**는 영문보다 넓은 너비로 계산 (12pt vs 6.5pt)
