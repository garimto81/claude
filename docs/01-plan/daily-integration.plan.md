# Plan: Daily Integration

> `/secretary` + Slack List 통합 - 개인 업무와 프로젝트 진행률을 단일 대시보드로

**Version**: 1.0.0
**Created**: 2026-02-05
**Status**: Planning
**PRD**: N/A (내부 개선)

---

## 1. 배경 및 문제 정의

### 1.1 현재 상태

두 개의 별개 시스템이 유사한 목적을 수행:

| 시스템 | 목적 | 데이터 소스 | 관점 |
|--------|------|------------|------|
| `/secretary` | 개인 업무 현황 | Gmail, Calendar, GitHub | 개인 중심 |
| Slack List | 프로젝트 진행률 | GitHub PRs, Checklists | 프로젝트 중심 |

### 1.2 문제점

```
┌─────────────────────────────────────────────────────────────┐
│  문제: 분산된 정보 → 통합 가시성 부재                        │
├─────────────────────────────────────────────────────────────┤
│  • "오늘 할 일" 확인하려면 → /secretary 실행                 │
│  • "프로젝트 진행률" 확인하려면 → Slack List 확인            │
│  • 두 정보의 연결고리 없음 (메일 ↔ 프로젝트)                │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 본질적 니즈

**"흩어진 정보를 한 곳에서 파악"** - 두 시스템 모두 같은 근본 문제 해결

## 4. 아키텍처 개요

### 4.1 통합 구조

```
┌─────────────────────────────────────────────────────────────┐
│                     /daily 스킬 (신규)                       │
│                                                             │
│  ┌─────────────────┐  ┌─────────────────┐                  │
│  │   /secretary    │  │   Slack List    │                  │
│  │   (기존 유지)    │  │   Reader (신규) │                  │
│  └────────┬────────┘  └────────┬────────┘                  │
│           │                     │                           │
│           ▼                     ▼                           │
│  ┌─────────────────────────────────────────────────┐       │
│  │              Daily Dashboard Formatter          │       │
│  │  • 개인 업무 섹션 (Gmail, Calendar, GitHub)     │       │
│  │  • 프로젝트 섹션 (PRD별 진행률)                 │       │
│  │  • 연결 섹션 (메일 ↔ PRD, PR ↔ Checklist)      │       │
│  └─────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 데이터 흐름

```
1. /daily 실행
   │
   ├── /secretary 스크립트 호출 (기존)
   │   └── Gmail, Calendar, GitHub 데이터 수집
   │
   ├── Checklist 파일 읽기 (신규)
   │   └── docs/checklists/*.md 파싱
   │
   └── 통합 출력 생성
       └── 개인 + 프로젝트 + 연결 정보
```

## 5. 구현 전략

### 5.1 Phase 1: 최소 통합 (MVP)

| 작업 | 설명 | 예상 시간 |
|------|------|:--------:|
| `/daily` 스킬 생성 | SKILL.md + 기본 구조 | 1h |
| secretary 호출 통합 | 기존 스크립트 재사용 | 0.5h |
| Checklist 파서 | `docs/checklists/*.md` 읽기 | 1h |
| 통합 포매터 | 대시보드 출력 형식 | 1h |

**MVP 출력 예시:**
```
📊 Daily Dashboard (2026-02-05)
═══════════════════════════════════════════════════

👤 개인 업무
├── 📧 이메일 할일: 3건 (긴급 1)
├── 📅 오늘 일정: 2건
└── 💻 GitHub 주의: PR #42 리뷰 대기 3일

📊 프로젝트 진행률
├── PRD-0001 ████████░░ 80% - 인증 시스템
├── PRD-0002 ██░░░░░░░░ 20% - PR 수집
└── PRD-0135 ██████░░░░ 60% - 워크플로우 활성화

═══════════════════════════════════════════════════
```

### 5.2 Phase 2: 서브커맨드 추가

| 서브커맨드 | 용도 |
|-----------|------|
| `/daily standup` | 아침 - 오늘 할 일 + 일정 |
| `/daily retro` | 저녁 - 오늘 완료 + 다음 계획 |
| `/daily projects` | 프로젝트 진행률만 |

### 5.3 Phase 3: 연결 정보 (선택)

- 메일 제목에서 PRD 번호 추출 → PRD와 연결
- PR 번호에서 Checklist 항목 매칭
