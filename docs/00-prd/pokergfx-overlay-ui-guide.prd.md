# PokerGFX 오버레이 UI 설명서 PRD

**버전**: v1.0 | **작성일**: 2026-02-27 | **기준 매뉴얼**: PokerGFX v3.2.0

---

## 1. 배경 및 목적

### 1.1 배경

PokerGFX는 포커 방송 제작에 특화된 그래픽 오버레이 소프트웨어다. v3.2.0 기준으로 15개 이상의 오버레이 요소, 다양한 스킨 시스템, 복잡한 설정 옵션을 제공하며, 공식 매뉴얼(113페이지)만으로는 실제 방송 현장에서의 빠른 참조가 어렵다.

현재 팀은 PokerGFX를 WSOP 방송 자동화 인프라(`C:\claude\automation_hub`)와 연동하여 운영 중이며, 각 오버레이 요소의 동작 방식, 레이아웃 규칙, 설정 옵션을 체계적으로 정리한 운영용 UI 설명서가 필요하다.

### 1.2 목적

- **운영 참조 문서**: 방송 제작 중 각 오버레이 요소의 설정 옵션과 동작을 빠르게 조회
- **온보딩 자료**: 신규 운영자가 PokerGFX의 주요 기능을 체계적으로 학습
- **설정 표준화**: 현재 활성 설정값(Titanium 스킨, Vert/Bot/Spill 레이아웃 등)을 문서화하여 재현 가능한 설정 기준 수립
- **통합 연동 가이드**: automation_hub와의 연동 시 각 오버레이 요소의 동작 조건을 명확히 정의

### 1.3 문서 범위

PokerGFX v3.2.0 매뉴얼(113페이지)과 실제 스크린샷 25장을 기반으로, 15개 오버레이 요소 각각에 대한 상세 설명서를 작성한다. 공식 매뉴얼 번역이 아닌, **운영 현장 중심의 실용 설명서** 형태로 구성한다.

## 2. 요구사항 목록

### 2.1 각 오버레이 요소 설명서에 포함해야 할 항목 (공통 구조)

각 오버레이 요소 설명서는 아래 항목을 반드시 포함해야 한다.

| # | 항목 | 내용 |
|---|------|------|
| R-01 | **개요** | 요소의 목적, 방송에서의 역할, 대표 스크린샷 참조 |
| R-02 | **화면 측정값** | 픽셀 크기(가로x세로), 4K 대응 여부 |
| R-03 | **구성 요소 목록** | 요소를 구성하는 서브 컴포넌트 열거 (예: 이름 바, 스택 바, 홀 카드 등) |
| R-04 | **레이아웃 모드** | 지원하는 레이아웃 종류와 각각의 시각적 차이 설명 |
| R-05 | **표시 조건** | 언제 화면에 나타나는지 (예: 핸드 시작, 액션 시, 올인 시) |
| R-06 | **공개 방식** | 요소 또는 서브 컴포넌트가 공개되는 타이밍 옵션 |
| R-07 | **트랜지션** | 등장/퇴장 애니메이션 종류와 설정값 (Slide/Fade/Pop/Expand) |
| R-08 | **설정 옵션 전체 목록** | 에디터에서 설정 가능한 옵션과 기본값/현재 활성값 |
| R-09 | **스킨 연동** | Titanium 스킨 기준 시각적 특성, 폰트, 색상 |
| R-10 | **동작 시나리오** | 실제 핸드 진행 중 요소가 동작하는 주요 시나리오 (텍스트 서술) |
| R-11 | **주의 사항 / 알려진 이슈** | 설정 충돌, 오버라이드 조건, 버전별 차이 등 |
| R-12 | **관련 요소 연동** | 다른 오버레이 요소와의 상호작용 (예: Player ↔ Board, Action Clock ↔ Player) |

### 2.2 요소별 추가 필수 항목

각 오버레이 요소의 특성에 따라 추가 항목이 필요하다.

| 오버레이 요소 | 추가 필수 항목 |
|--------------|--------------|
| Player Graphic | 카드 공개 방식(6종), 폴드 표시 방식, 사진 업로드/촬영 방법, Mixed Games 카드 수 설정, Indent/Bounce 애니메이션, Hilite 색상 설정 |
| Board Graphic | 커뮤니티 카드 슬롯 구성, POT/사이드팟 표시 규칙, 위닝 핸드 하이라이트 조건(4종), Vanity 텍스트 설정 |
| Blinds Graphic | 블라인드 표시 조건(4종), Ante/Button Blind 처리, 블라인드 레벨 전환 규칙 |
| Leaderboard | 동적 리사이즈 방식, 표시 통계 항목 선택(VPIP/PFR/AGRfq 등), 스폰서 로고 배치, 탈락 플레이어 색상 구분 |
| Ticker | 자동 삽입 통계 설정, 스크롤 방향/속도, 루프 옵션 |
| Strip | 정렬 기준(Chip Count/Table Position), 동적 리사이즈 규칙, 탈락 플레이어 표시 옵션 |
| Outs Graphic | True Outs vs Theoretical Outs 차이 설명, 머크 카드 포함/제외 동작 |
| Action Clock | 카운트다운 표시 임계값, 외부 태블릿 앱 연동 방법 |
| Heads-Up History | 베팅 라운드별 섹션 구조, 동적 리사이즈 조건 |
| PIP | 토너먼트 클럭 화면 캡처 방법, 표시/숨김 트리거 |
| Cards | 4색 카드 세트 설정, 게임 변형별 오버라이드 방법, Rabbit 오버레이 동작 |
| Countdown | Secure Delay 설정값(기본 30분), 리드아웃 비디오 삽입 방법 |

| Board Position | Right |

| Reveal Players | Action On |
| Reveal Cards | Immediate |
| 에쿼티 표시 | 1st 베팅 라운드 이후 |

| 위닝 핸드 하이라이트 | Immediately |

### 3.4 범위 외 (OUT OF SCOPE)

- PokerGFX 서버 설치/초기 설정 방법 (별도 운영 매뉴얼 대상)

## 4. 비기능 요구사항

### 4.1 문서 형식

| 항목 | 기준 |
|------|------|
| **언어** | 한글 작성, 기술 용어(옵션명, UI 레이블, 파일 포맷)는 영어 원문 유지 |
| **파일 형식** | Markdown (`.md`), GitHub 렌더링 기준 |
| **다이어그램** | 저장 파일 내 Mermaid 코드 블록 사용 (ASCII 금지), 노드 줄바꿈은 `<br/>` 사용 |
| **UI 목업** | `--mockup` 패턴으로 HTML 목업 생성 (B&W Refined Minimal 기본, ASCII 와이어프레임 금지) |
| **이미지 참조** | 스크린샷은 `![설명](경로)` 형식으로 인라인 참조, 번호 매핑 테이블 제공 |
| **코드 블록** | 설정값, 파일 경로는 `` `코드` `` 형식 사용 |

### 4.2 구성 기준

| 항목 | 기준 |
|------|------|
| **요소별 분리** | 15개 오버레이 요소 각각을 독립적인 섹션 또는 독립 파일로 작성 |
| **설정 옵션 표** | 모든 설정 옵션은 표 형식으로 정리 (옵션명 / 값 종류 / 기본값 / 현재 활성값) |
| **시나리오 서술** | 동작 시나리오는 단계별 번호 목록(1. 2. 3.) 또는 표로 작성 |
| **상호 참조** | 관련 요소 간 섹션 링크(`[Board Graphic](#board-graphic)`) 필수 |

### 4.3 스크린샷 활용

| 스크린샷 그룹 | 활용 목적 |
|-------------|---------|
| 서버 설정 UI 7장 | 각 탭(System/Sources/Outputs/GFX1/GFX2/GFX3/Commentary) 설명 보조 |
| 그래픽 에디터 10장 | 각 요소 에디터 화면과 측정값 참조 |
| 스킨 에디터 1장 | Titanium 스킨 구성 요소 목록 설명 보조 |
| Outputs 설정 4장 | box7 UI 행 측정값(권장 280px vs 현재 800px) 등 실측 근거 제공 |

## 6. 우선순위

### 6.1 구현 단계

```
Phase 1 (핵심 요소)     → Player Graphic, Board Graphic, Leaderboard
Phase 2 (운영 필수)     → Blinds, Ticker, Strip, Action Clock, Outs, Cards
Phase 3 (부가 요소)     → Field, Heads-Up History, Split Screen, PIP, Commentary Header, Countdown
Phase 4 (스킨 시스템)   → Titanium 스킨, 커스텀 애니메이션, Z-order, Anchor, 4K, 크로마 키
```

### 6.2 우선순위 근거

| 우선순위 | 요소 | 근거 |
|:--------:|------|------|
