## 1. 개요

### 1.1 배경

WSOP Automation 프로젝트는 4개의 서브 프로젝트(Hub, FT, AE, SUB)로 구성되어 있으며, 전체 완료에 108-128일이 필요합니다. 그러나 가용 작업일은 28일뿐이므로, 핵심 기능(MVP)만 우선 완료하는 전략이 필요합니다.

### 1.2 목적

- 4개 프로젝트의 MVP 범위 정의 및 일정 관리
- 주차별/일별 진행 상황 추적
- 마일스톤 달성 여부 모니터링
- 리스크 식별 및 대응 계획 수립

### 1.3 범위

| 포함 | 제외 |
|------|------|
| MVP 일정 정의 | 전체 기능 일정 |
| 진행률 추적 (Markdown) | 웹 대시보드 |
| 마일스톤 관리 | 자동화된 알림 |
| 리스크 관리 | Slack 연동 |

## 2. 프로젝트별 MVP 정의

### 2.1 automation_hub (12일)

**MVP 범위:**
- PRD-0005 App1/2/3 (Production 워크플로우)
- PRD-0003 Phase 1~2 (모델 확장 + 어댑터)

**제외 항목:**
- PRD-0004 (프론트엔드 대시보드)
- PRD-0001/0002 (인프라/모니터링)

**MVP 산출물:**
```
GFX JSON → DB → 데이터 선택 → AE 렌더링 전체 흐름 동작
```

### 2.2 automation_feature_table (8일)

**MVP 범위:**
- NAS SMB 연동 완료
- 기본 핸드 저장 (hands 테이블만)
- 실환경 테스트 (Primary 정확도)

**제외 항목:**
- PRD-0005 DB 전체 (21개 테이블)
- 자막 내보내기 (20종)
- Secondary 정확도 검증

**MVP 산출물:**
```
PokerGFX JSON → NAS → DB 저장 동작
```

### 2.3 automation_ae (5일)

**MVP 범위:**
- Phase 8 마무리 (배치 렌더링)
- 기존 렌더링 파이프라인 안정화

**제외 항목:**
- PokerGFX API 전체
- CyprusDesign 분석

**MVP 산출물:**
```
템플릿 → Nexrender → 렌더링 출력 안정 동작
```

### 2.4 automation_sub (3일)

**MVP 범위:**
- WSOP+ CSV 형식 정의
- 수기 입력 데이터 구조 정의
- Supabase 마이그레이션 적용

**제외 항목:**
- Backend/Frontend 구현
- 337개 항목 전체 구현
- Agent 아키텍처

**MVP 산출물:**
```
데이터 처리 방식 문서화 + DB 스키마 적용
```
